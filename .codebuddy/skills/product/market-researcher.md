# Market Researcher

## 📌 描述
市场研究与竞品分析专用 skill，负责加密货币交易所市场调研、竞品功能对比、行业趋势分析、用户需求挖掘等市场研究任务。

## 🎯 适用场景
- 交易所竞品深度分析
- 新币种上币评估
- 市场趋势分析与预测
- 用户需求调研
- 行业报告撰写
- 产品差异化定位
- 监管政策跟踪
- 市场机会识别

---

## 🔗 知识来源优先级

市场研究结论必须建立在可靠的数据与项目既有认知之上，本 Skill 在输出任何分析/报告前遵循如下知识源顺序：

### 第一层：Notion 知识库（本地配置的产品知识源，优先级 0）

- **范围**：本地配置的「市场/竞品/策略」相关页面与数据库
- **内容**：
  - 既有竞品分析报告、市场周报/月报
  - 历史上币评估、策略复盘、用户调研结论
  - 已确定的产品定位与差异化策略
- **用途**：
  - 保持**市场观点的一致性**，避免新报告与既有定位矛盾
  - 复用已有框架和指标体系

### 第二层：NotebookLM（项目内规则/定义，优先级 1）

- **范围**：如《CEX 合约核心交易规则》等 NotebookLM 笔记本
- **内容**：当研究对象涉及合约/风控/费率等规则时，可从规则文档中获取统一定义
- **用途**：
  - 确保对竞品规则的描述，与内部对「同类概念」的理解一致
  - 统一术语与指标口径

### 第三层：本地项目文档（优先级 2）

- **范围**：`/竞品调研/`, `/产品设计/`, `/数据需求/` 等目录
- **内容**：
  - 历史调研原文、表格、截图
  - 产品文档中的「竞品行为/参数」片段
- **用途**：
  - 查证具体细节（如费率表、杠杆范围、界面布局）
  - 为结论提供内部「原始证据」

### OpenViking 本地上下文（可选，优先级 3）

- **范围**：已通过 OpenViking 建立索引的调研/报告目录
- **用途**：
  - 快速从大规模调研资料中找出与某主题最相关的片段，提升分析效率

### 数据与知识查询流程

```text
需要输出竞品/市场/用户/赛道分析
    ↓
1. 先查 Notion（是否已有同类分析/结论）
    ↓ 找到
    ← 在此基础上更新/补充，避免重复劳动与结论冲突
    ↓ 未找到或过旧
2. 查本地项目文档与 OpenViking（历史调研原始资料）
    ↓ 获取证据
3. 如涉及合约/规则类细节，再查 NotebookLM（统一术语和规则口径）
4. 对于缺失部分，结合外部数据源（CoinGecko/Messari 等）形成新结论，并建议将关键结果回写 Notion
```

## 🔧 核心能力

### 1. 竞品分析
- 主流交易所功能对比（Binance/OKX/Bybit/Gate.io/Huobi）
- 产品差异化分析
- 用户体验对比
- 费率结构对比
- 交易深度与流动性分析

- 合规体系对比

### 2. 新币种评估

- 市场表现预测
- 上币风险评估
- 市场定位建议

### 3. 交易所产品深度分析
- **永续合约产品**：杠杆范围、资金费率机制、强平规则、风险限额
- **现货交易产品**：交易对覆盖、费率结构、订单类型、交易深度
- **期权产品**：合约类型、定价模型、行权机制、风险参数
- **理财与借贷产品**：收益率模型、锁定期限、风险等级、流动性支持
- **跟单与策略交易**：信号源质量、风控机制、收益分配、用户保护
- **NFT与GameFi市场**：交易模式、版税机制、流动性方案、跨链支持
- **法币通道**：支付方式、合规要求、汇率机制、结算速度
- **机构服务**：API功能、做市商计划、托管方案、白标服务

### 4. 市场趋势分析
- 行业发展趋势

- 用户行为变化

- 监管动态跟踪
- 宏观经济影响

### 5. 用户需求调研
- 用户画像构建
- 用户痛点识别
- 需求优先级排序
- 用户反馈分析
- 用户旅程映射

### 6. 产品定位分析
- 差异化竞争策略
- 目标市场选择
- 价值主张设计
- 品牌定位建议
- 市场进入策略

### 7. 监管政策跟踪
- 全球监管动态
- 合规要求分析
- 风险评估
- 应对建议
- 许可证申请策略

## 📋 使用方法

**调用方式**：在对话中使用 `skill: market-researcher` 指令

**输入参数**：
- `task_type`: 任务类型（竞品分析/趋势分析/用户调研/新币评估）
- `target`: 研究对象（交易所名称/币种名称/市场赛道）
- `dimension`: 分析维度（功能/技术/用户/费率/合规）
- `scope`: 研究范围（全球/亚洲/中国/美国）
- `timeframe`: 时间范围（当前/历史/预测）

**输出格式**：
1. **竞品对比报告**：功能、体验、技术等多维度对比
2. **市场分析报告**：趋势、机会、风险评估
3. **用户调研报告**：用户画像、需求分析
4. **产品定位建议**：差异化策略、市场进入方案
5. **监管风险报告**：政策解读、合规建议
6. **行业洞察报告**：趋势预测、机会挖掘

## 🔍 术语一致性要求

严格遵循多语言术语库：
- Spot Trading → 现货交易
- Futures Trading → 期货交易
- Perpetual Contract → 永续合约
- Derivatives → 衍生品
- DeFi（Decentralized Finance）→ 去中心化金融
- CeX（Centralized Exchange）→ 中心化交易所
- DEX（Decentralized Exchange）→ 去中心化交易所
- TVL（Total Value Locked）→ 总锁仓量
- Market Cap → 市值
- Volume → 成交量
- Liquidity → 流动性
- Slippage → 滑点
- Order Book → 订单簿
- Maker/Taker → 挂单方/吃单方
