## Browser Contract

- `iab` means Codex in-app Browser plugin. Use it for public pages, Chinese smoke, market display, and stateless regression.
- `chrome` means the user's Chrome plugin/session. Use it only when the flow needs existing login state or account-specific pages.
- Python scripts in this Skill do not inspect Chrome cookies, local storage, profile data, passwords, or session stores.
- Browser/Chrome plugins are the live execution surfaces. Python scripts provide flow linting, dry-run execution, report assembly, route resolution, and non-sensitive precheck helpers.

## Selector Policy

Selector priority:

1. `testid` (`data-testid`)
2. `css`
3. exact `text`
4. `text_regex`
5. `role` + `name`
6. `fallback_xy` as temporary compatibility

New Web surfaces should expose stable `data-testid` values. Coordinate fallback must be treated as an automation risk and promoted to a test-id request through the learning loop.

## Account Profiles

Supported profiles are `guest`, `basic-login`, `funded-spot`, `funded-futures`, `open-orders`, `open-position`, and `withdraw-capable`. Flow profile mismatch is an `environment_blocker`, not a product bug.

## Trading Safety

- Default trade flows stop before final submit.
- Testnet final submit requires flow permission, `--allow-side-effects`, `--confirm-submit`, and a step-level `safety_gate`.
- Withdrawal flows are probe-only in the first version; do not execute real withdrawal submission.
- All key trade conclusions require page-state plus result-state verification.
