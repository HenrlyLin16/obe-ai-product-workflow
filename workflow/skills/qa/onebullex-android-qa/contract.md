## Locator Policy

Selector priority:

1. `resource-id`
2. `content-desc`
3. exact `text`
4. regex `text` or `content-desc`
5. coordinate fallback

New Android surfaces should expose stable IDs. For Compose screens, enable
`testTagsAsResourceId = true` at the app root and set stable `testTag` values.
Use automation IDs for testing contracts and reserve human `contentDescription`
for accessibility where possible.

## Route Map

Reusable page controls live in `routes/*.yaml`. Flows should reference route
keys with `route: feature.element` instead of repeating selector and coordinate
fallback data in every step.

Route entries may contain:

- `selector`: uiautomator selector used by `tap`, `input`, or wait-like actions
- `coordinate`: temporary fallback coordinate for `tap_xy`
- `description`: human-readable context
- `stability`: `stable`, `text`, `fallback`, or `unknown`
- `needs_test_id`: true when Android should add a stable automation ID

When a flow run uses coordinate fallback, the learning loop should generate a
route update candidate rather than treating the coordinate as a permanent
solution.

## Safety

Login credentials must come from environment variables. Trading or order flows
must stop before final side effects unless the user explicitly confirms account,
environment, symbol, direction, quantity, leverage, and final action.
