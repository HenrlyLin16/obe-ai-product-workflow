# OBE 产品 / 交互体验自查清单（单一事实源）

**用途**：产品方案、交互方案、原型方案定稿或评审前的 **体验与界面质量门**；可与 PRD / 规则文档并行使用（规则准确性仍以 `web3-product-expert`、`perpetual-designer` 及 Notion 为准）。

**版本**：v1.2  
**最后更新**：2026-05-02

## 设计参考与品味（提升自动化判断质量）

品味作 **模式识别**：外部参考须 **结构化进交付物**，否则 Agent 只会泛泛「更美观」。**执行手册**：[OBE-design-reference-playbook.md](OBE-design-reference-playbook.md)（分层资源：上线产品 / Mobbin / Figma Community / Dribbble 边界 / 设计思维读物；含 PRD「参考样本索引」模板与 Agent 指令）。

## 来源与对齐说明

| 来源 | 说明 |
|------|------|
| CoinStore《用户体验自查表》 | 本地表格「任务流程—框架布局—页面交互—页面元素—组件—运营—特殊场景」七大块，已 **去排版噪声、合并重复维度、标注优先级**，并改写为可勾选条目 |
| OneBullEX 交互体验基线 | 在前表基础上增加 **Web3 CEX / 永续合约** 专用的风险展示、数值口径、实时性与权限门禁 |

**优先级约定**

- **P0**：缺失则不建议定稿或上线评审（团队质量门必查）。
- **P1**：建议在 MVP 或迭代早期补齐；可记录在 PRD「体验债」。
- **P2**：按资源安排优化。

## 在工作流中的挂载（与团队流水线对齐）

以下流水线在 `pipelines.yaml` 中已配置 **`mandatory_references` / `quality_gates`**，须 **显式引用本文档**：`prd-full`、`prd-and-qa`、`futures-feature`、`ai-feature`、`ux-audit`、`prototype-full`。

| 阶段 | 建议动作 | 对应流水线（见 `pipelines.yaml`） |
|------|----------|-----------------------------------|
| 概念 / 信息架构冻结 | 完成 **§1 任务流程** + **§8 Web3 附加** 中与流程相关的 P0 | `prd-full`、`futures-feature` 早期步骤 |
| PRD / 交互说明评审 | 完成 **§2～§5** 中与本次范围相关的全部 P0 + 关键 P1；PRD 内记录检查 ID | `prd-full`、`prd-and-qa`、`futures-feature`（默认 §8 P0 必经，除非 PRD 声明纯后端无 UI 并经评审确认）、`ux-audit` |
| 原型 / Figma / H5 冻结 | 全表 P0 + 与页面相关的 P1；**§8** 全 P0 | `prototype-full`（初版与修订后各至少一轮 P0） |
| 交付前团队质量门 | `web3-cex-product-team.md` 原有条目 **+ 本节 P0 抽检或全检** | 全部团队流水线 |

**协作 Skill**：交互与原型产出由 `web3-product-experience-expert` 主导自检；主导产品 Skill 负责 **§8** 与业务规则一致性抽检。

## §1 任务流程

| ID | 自查点 | 检查内容（过关标准） | 优先级 |
|----|--------|----------------------|--------|
| UX-F-01 | 通畅性 | 主路径可跑通；无死循环、无「断崖」（无前因后果的跳转） | P0 |
| UX-F-02 | 可逆性 | 用户能回答「从哪来、回哪去」；返回栈与 deep link 一致 | P0 |
| UX-F-03 | 撤销 / 重置 | 关键流程是否支持撤销、重置或明确告知不可撤销及后果 | P0 |
| UX-F-04 | 退出 | 流程可随时退出；退出方式（手势/按钮/系统返回）说明完整 | P1 |
| UX-F-05 | 可续性 | 长流程支持保存 / 草稿 / 自动保存；意外退出有恢复或提示 | P1 |
| UX-F-06 | 权限与分支 | 不同权限是否走不同分支；禁用时是否解释原因与获取方式 | P0 |
| UX-F-07 | 时效 | 流程是否过期（验证码、支付倒计时、活动）；失效后的引导 | P1 |
| UX-F-08 | 容错与不通过 | 校验失败、风控拒绝、余额不足等有明确下一步，非空白失败 | P0 |
| UX-F-09 | 流程耦合 | 与本需求相关的其他流程是否冲突（并发下单、模式切换等） | P0 |
| UX-F-10 | 特殊说明 | 易误解环节是否有简短说明或示例（不过度依赖外链帮助） | P1 |

## §2 框架与布局

| ID | 自查点 | 检查内容 | 优先级 |
|----|--------|----------|--------|
| UX-L-01 | 导航 | 导航易触达、可扩展；模式（底栏 / 顶栏 / Tab / 汉堡）与场景匹配 | P1 |
| UX-L-02 | 内容类型 | 页面内容类型完备列出；特殊类型（衍生品卡片、风险提示条）单独约定 | P1 |
| UX-L-03 | 主次层级 | 重要信息强化，次要信息弱化；交易页核心数字不被装饰淹没 | P0 |
| UX-L-04 | 操作区 | 高频操作落在易触达区；单手 / 大屏场景抽样自检 | P1 |
| UX-L-05 | 合并 / 拆分 | 是否需要拆分步骤或合并视图以降低跳转（如下单确认合一页） | P2 |

| ID | 自查点 | 检查内容 | 优先级 |
|----|--------|----------|--------|
| UX-LI-01 | 排布 | 分页或一次性加载策略明确；分页容量与步骤条（若有）一致 | P0 |
| UX-LI-02 | 元素规格 | 单列元素的 **状态 / 前置与后置条件 / 关联变化** 在 PRD 或标注中写清 | P0 |
| UX-LI-03 | 排序 | 默认排序与规则可见；支持切换时反馈一致 | P1 |
| UX-LI-04 | 交互反馈 | 点击 / 悬停有反馈；说明反馈是展开、跳转还是抽屉 | P0 |
| UX-LI-05 | 折叠与筛选 | 长列表是否有折叠、筛选以降低认知负荷 | P1 |
| UX-LI-06 | 负向操作 | 删除、隐藏、取消订阅支持情况及二次确认策略 | P1 |
| UX-LI-07 | 空状态 | 列表为空有引导或情感化兜底 copy（含下一步） | P0 |
| UX-LI-08 | 网络状态 | 弱网 / 断网有提示与重试；与 §7 网络条目一致 | P0 |
| UX-LI-09 | 列表尽头 | 到底是否有提示（无更多 / 加载更多） | P2 |

## §4 页面级交互（加载 / 刷新 / 转场）

| ID | 自查点 | 检查内容 | 优先级 |
|----|--------|----------|--------|
| UX-P-01 | 入场 / 转场 | 是否有约定转场；快速返回手势是否与系统一致且不冲突 | P2 |
| UX-P-02 | 加载骨架 | 是否有骨架屏或预设占位；避免布局跳动 | P1 |
| UX-P-03 | 等待过长 | 长请求是否有进度、分步或预估；能否取消 | P1 |
| UX-P-04 | 加载失败 | 失败态可见；自动重试或手动重试；多次失败有兜底（客服 / 反馈入口） | P0 |
| UX-P-05 | 刷新策略 | 自动 / 手动；下拉或按钮；切换 Tab / 页面是否需主动刷新写清 | P0 |
| UX-P-06 | 刷新增量 | 每次刷新条数；是否提示「新消息数量」等 | P1 |
| UX-P-07 | 刷新空结果 | 无新内容时的提示（避免静默空白） | P0 |

## §7 组件与控件

### 7.1 按钮与反馈

| ID | 自查点 | 检查内容 | 优先级 |
|----|--------|----------|--------|
| UX-B-01 | 类型与层级 | 主 / 次 / 幽灵 / 文字按钮与信息层级匹配 | P1 |
| UX-B-02 | 位置 | 易触达； destructive 与 primary 不抢戏错误 | P0 |
| UX-B-03 | 状态 | 默认态、加载态、禁用态齐全；禁用原因可获知或有 Tooltip | P0 |
| UX-B-04 | 反馈强度 | 成功 / 失败 / 无权限 / 空状态选用合适载体（弹窗 / Toast / 行内提示） | P0 |
| UX-B-05 | 操作前后变化 | 关键操作后界面状态变化可见、可验证 | P0 |

### 7.2 弹窗与破坏性操作

| ID | 自查点 | 检查内容 | 优先级 |
|----|--------|----------|--------|
| UX-M-01 | 二次确认 | 核心 / 毁灭性操作（平仓、转模式、删除 API Key）有确认 | P0 |
| UX-M-02 | 遮罩与关闭 | 点击遮罩是否关闭约定一致；避免误触关闭导致损失 | P1 |
| UX-M-03 | 行动点 | 主按钮突出；destructive 标红；保留取消路径 | P0 |

### 7.3 输入与表单

| ID | 自查点 | 检查内容 | 优先级 |
|----|--------|----------|--------|
| UX-IN-01 | 类型与限制 | 字符集、长度、精度说明；密码等敏感输入可隐藏 | P0 |
| UX-IN-02 | 过载展示 | 超长输入截断或滚动策略 | P1 |
| UX-IN-03 | 提示与预填 | placeholder、默认值、错误原因与修复建议 | P0 |
| UX-IN-04 | 报错 | 错误样式显著；文案说明如何改正；焦点可落到错误字段 | P0 |
| UX-IN-05 | 报错强度与频次 | 合适强度；频繁报错是否有解锁路径（人工 / 冷却） | P1 |

### 7.4 标签栏 / 单选多选

| ID | 自查点 | 检查内容 | 优先级 |
|----|--------|----------|--------|
| UX-T-01 | 默认与必选 | 默认选项；必填标识 | P0 |
| UX-T-02 | 操作 | 支持清空或撤销选择（若业务允许） | P2 |

## §8 Web3 CEX / 永续合约附加（体验 × 资损）

**说明**：本节与业务规则 PRD 互补；数值公式以 Notion / `perpetual-designer` 为准，本节只管 **「用户看见什么、何时看见、错了会怎样」**。

| ID | 自查点 | 检查内容 | 优先级 |
|----|--------|----------|--------|
| UX-W3-01 | 价格口径 | 展示价 vs 触发价（标记 / 最新 / 指数）一致或可解释；切换时有说明 | P0 |
| UX-W3-02 | 核心数字 | 保证金、可用、盈亏、杠杆、强平价：单位、符号、精度与后端一致 | P0 |
| UX-W3-03 | 风险前置 | 开仓 / 调杠杆 / 模式切换前可见关键风险摘要（非仅此一页免责） | P0 |
| UX-W3-04 | 实时与延迟 | 行情 / 持仓延迟或 stale 时有提示；避免用户基于过期数字决策 | P0 |
| UX-W3-05 | 极端行情 | 熔断、无法平仓、仅减仓等模式的界面状态与文案有预案 | P0 |
| UX-W3-06 | 费用与滑点 | 手续费、资金费、预估成交价差在中等及以上风险路径可见或可展开 | P1 |
| UX-W3-07 | 不可逆操作 | 平仓、合并仓位、切换全仓逐仓等：确认文案含后果摘要 | P0 |

## §10 特殊场景与环境

| ID | 自查点 | 检查内容 | 优先级 |
|----|--------|----------|--------|
| UX-X-01 | 横竖屏 | 是否支持横屏；布局与核心操作完整性 | P2 |
| UX-X-02 | 系统差异 | iOS / Android / Web 行为差异是否列出（返回键、安全区等） | P1 |
| UX-X-03 | 权限 | 相机、相册、通知等：用途说明与授权引导 | P1 |
| UX-X-04 | 深色模式 | 是否需要；对比度与语义色在深色下可读 | P2 |
| UX-X-05 | 省流 / 低电量 | 自动刷新、大图、视频是否在弱网 / 低电量下降级 | P2 |

| UX-X-10 | 高峰降级 | 业务高峰是否有前端降级展示（缓存数据标识、排队提示） | P1 |

## §11 参考源与对照样本（品味 × 约束）

| ID | 自查点 | 检查内容 | 优先级 |
|----|--------|----------|--------|
| UX-REF-01 | 上线对照 | 至少 **1 例**「已上线」同类产品流程或屏（竞品客户端 / [Mobbin](https://mobbin.com/discover) 同类功能）；PRD 或交互文档中有 **参考样本索引** 表（见 Playbook §3.1） | P2 |
| UX-REF-02 | 展示型边界 | 若使用 [Dribbble](https://dribbble.com/) 等概念稿，是否标明 **仅视觉/局部灵感**，并写明 **因真实约束而不采纳** 的点（加载、权限、数据时效等） | P2 |
| UX-REF-03 | 高密度可读（B2B） | 复杂表单或仪表盘是否可对齐「高密度可读」基准（如 Linear / Vercel Dashboard 一类 **信息层级**，非抄袭视觉）；取舍是否写入 PRD | P2 |
