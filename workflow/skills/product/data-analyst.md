# Data Analyst

## 📌 描述
交易数据分析专用 skill，负责加密货币交易所交易数据的深度分析、可视化、指标计算和报告生成，为产品决策和风控优化提供数据支持。

## 🎯 适用场景
- 交易数据分析
- 用户行为分析
- 市场深度分析
- 风控指标分析
- 业务报表生成
- 数据可视化
- 趋势预测分析
- 异常检测

---

## 🔗 知识来源优先级

为保证所有分析结果与项目统一指标口径和规则一致，本 Skill 在设计/计算指标时遵循以下知识源顺序：

### 第一层：Notion 知识库（项目内部指标定义，优先级 0）

- **范围**：数据指标定义、埋点规范、业务报表说明等页面
- **内容**：
  - 指标口径定义（如强平率/穿仓率/交易量等）
  - 埋点/事件与字段列表
  - 现有报表结构与监控大盘
- **用途**：
  - 作为「**指标定义的唯一真相来源**」
  - 避免同名指标在不同分析中含义不一致

### 第二层：NotebookLM（规则与公式背景，优先级 1）

- **范围**：如《CEX 合约核心交易规则》等 NotebookLM 笔记本
- **内容**：与永续合约/Futures 规则相关的量（保证金、强平价、资金费等）
- **用途**：
  - 当分析内容依赖这些规则（例如「强平价偏离」「资金费用率」等）时，用于获取**正确的公式与变量含义**

### 第三层：本地项目文档（优先级 2）

- **范围**：`/数据需求/`, `/产品设计/`, `/合约/` 等目录
- **内容**：
  - 业务方/产品给出的数据需求文档
  - 系统/表结构设计、字段定义
  - 历史分析报告
- **用途**：
  - 对齐数据库字段与接口返回字段
  - 理解数据来源链路与加工逻辑

### OpenViking 本地上下文（可选，优先级 3）

- **范围**：被 OpenViking 索引的历史报表、分析、需求文档
- **用途**：
  - 在做新分析前查找「是否有类似主题的既有分析」，提高复用度

### 指标与数据查询流程

```text
设计/计算数据指标或撰写分析报告
    ↓
1. 查 Notion（指标定义 / 埋点规范 / 报表说明）
    ↓ 找到
    ← 以其口径为准设计计算逻辑
    ↓ 未找到或不全
2. 查本地项目文档与 OpenViking（数据需求 / 表结构 / 历史分析）
    ↓ 补充字段与业务背景
3. 如指标依赖合约规则（如强平/资金费），查询 NotebookLM 获取正确公式
4. 若仍无权威定义：
    - 明确标注「临时定义」（建议后续在 Notion 中固化）
    - 避免在不同文档中使用不同临时定义
```

## 📋 使用方法

**调用方式**：在对话中使用 `skill: data-analyst` 指令

**输入参数**：
- `task_type`: 任务类型（交易分析/用户分析/风控分析/市场分析）
- `data_source`: 数据来源（数据库/API/文件）
- `time_range`: 时间范围
- `symbol`: 交易对（如 BTCUSDT）
- `metrics`: 指标列表
- `output_format`: 输出格式（表格/图表/报告）

**输出格式**：
1. **数据报表**：结构化的数据汇总
2. **分析报告**：深度分析与洞察
3. **可视化图表**：数据图表展示
4. **预测模型**：趋势预测结果
5. **异常报告**：异常检测结果

## 🔍 数据分析规范

### 1. 数据准确性
- 数据来源必须明确标注
- 计算公式必须清晰说明
- 异常值必须处理或标注
- 缺失值处理方法需说明

### 2. 分析方法
- 统计方法需说明
- 对比基准需明确
- 趋势判断需有数据支撑
- 结论需有充分证据

### 3. 可视化规范
- 图表标题清晰
- 坐标轴标注完整
- 图例说明清楚
- 颜色搭配合理
- 数据标签明确

### 4. 报告结构
- **摘要**：核心发现
- **数据概览**：基本统计
- **详细分析**：深度解读
- **趋势预测**：未来展望
- **建议**：行动建议

### 5. 术语一致性
严格遵循多语言术语库：
- Volume → 成交量
- Turnover → 成交额
- Order Book → 订单簿
- Depth → 深度
- Spread → 价差
- Slippage → 滑点
- Volatility → 波动率
- Liquidation Rate → 强平率
- Bankruptcy Rate → 穿仓率
- Risk Rate → 风险率

## 🔧 Python 代码示例

### 基础数据分析
```python
import pandas as pd
import numpy as np

# 读取数据
df = pd.read_csv('trading_data.csv')

# 基础统计
print(df.describe())

# 计算指标
df['pct_change'] = df['close'].pct_change() * 100
df['volatility'] = df['pct_change'].rolling(window=20).std()

# 相关性分析
correlation = df[['volume', 'close', 'volatility']].corr()
print(correlation)
```

### 时间序列分析
```python
from statsmodels.tsa.seasonal import seasonal_decompose

# 季节性分解
result = seasonal_decompose(df['close'], model='additive', period=24)

# 可视化
result.plot()
```

### 异常检测
```python
from sklearn.ensemble import IsolationForest

# 异常检测
model = IsolationForest(contamination=0.05)
df['anomaly'] = model.fit_predict(df[['close', 'volume']])

# 标记异常
anomalies = df[df['anomaly'] == -1]
print(f"检测到 {len(anomalies)} 个异常点")
```
